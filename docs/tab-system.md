# Tab System

Keeper supports a browser-style tab system that lets users keep multiple notes open simultaneously within the editor screen. Tabs are **not** a navigation-layer concept (not React Navigation tabs) -- they are a UI feature scoped to the editor screen, managed by a Zustand store and kept in sync with the router URL.

## Architecture Overview

```
┌─────────────────────────────────────────────────────┐
│  Editor Screen  (app/editor.tsx)                     │
│  ┌───────────────────────────────────────────────┐  │
│  │  TabBar  (components/TabBar.tsx)              │  │
│  │  [📌 Note A ✕]  [Note B ✕]  [📌 Note C ✕]    │  │
│  └───────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────┐  │
│  │  NoteEditorContent                            │  │
│  │  (HybridEditor + blocks)                      │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
          ▲                        ▲
          │  useTabStore           │  router.push/replace
          │  (state + actions)     │  `/editor?id=<noteId>`
          └────────────────────────┘
                    Tab Store + URL kept in sync
```

## Data Model

### `Tab` Interface

```ts
interface Tab {
  id: string;         // Unique ID generated by nanoid()
  noteId: string;     // The note this tab is viewing
  title: string;      // Display title (auto-synced from note metadata)
  isPinned: boolean;  // Pinned tabs cannot be closed via the UI
}
```

### `TabState` Interface

| Field | Type | Description |
|-------|------|-------------|
| `tabs` | `Tab[]` | All open tabs |
| `activeTabId` | `string \| null` | ID of the currently active tab |

### Actions

| Action | Parameters | Behavior |
|--------|------------|----------|
| `openTab` | `noteId`, `title?` | Opens a new tab. If a tab for `noteId` already exists, activates it instead (no duplicates). |
| `closeTab` | `tabId` | Removes the tab. No-op if pinned. When closing the active tab, prefers the **right neighbor**, then **left neighbor**, then `null`. |
| `activateTab` | `tabId` | Sets `activeTabId`. Silently ignores unknown IDs. |
| `pinTab` | `tabId` | Toggles the `isPinned` flag. |
| `updateTabTitle` | `tabId`, `title` | Updates a tab's display title. |
| `closeAllUnpinned` | — | Closes all unpinned tabs; activates the first pinned tab if any remain. |

## Core Store

**File:** `src/stores/tabStore.ts`

The tab state is managed in a single Zustand store. Key design decisions:

- **No persistence** -- tabs reset on app restart.
- **No duplicate tabs** -- `openTab` checks for an existing tab with the same `noteId` and activates it instead of creating a new one.
- **Safe close logic** -- `closeTab` is a no-op for pinned tabs. When the active tab closes, it activates the right neighbor first (like browser tabs), falling back to the left neighbor, then `null`.

## UI Component: TabBar

**File:** `src/components/TabBar.tsx`

A horizontal `ScrollView` of tab "chips" rendered at the top of the editor screen.

### Behavior

- **Mobile visibility**: Hidden on native platforms (`Platform.OS !== "web"`) when there is **1 or fewer** tab.
- **Tab chip**: Displays a pin icon (if pinned), the title (truncated to 1 line), and a close "✕" button (hidden for pinned tabs).
- **Tap**: Activates the tab and navigates via `router.replace(/editor?id=${noteId})`.
- **Long press**: Toggles the tab's pinned state.
- **Close button**: Calls `closeTab`, then reads the store's new `activeTabId` to decide where to navigate next (next tab's note, or `/` home if no tabs remain).

### Styling

- Active tab: elevated background (`theme.colors.card`), solid border, bold text.
- Inactive tab: transparent background, no border, muted text.

## Editor Integration

**File:** `src/app/editor.tsx`

### Note Resolution

The editor determines which note to show using a **URL-first, tab-fallback** strategy:

```ts
const noteId =
  typeof params.id === "string" && params.id
    ? params.id                // URL param takes priority
    : (activeTab?.noteId ?? "");  // fallback to active tab
```

### Title Sync

When a note loads, a `useEffect` pushes the note's current title into the tab store:

```ts
useEffect(() => {
  if (activeTabId && note?.title) {
    updateTabTitle(activeTabId, note.title);
  }
}, [activeTabId, note?.title, updateTabTitle]);
```

### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Meta+T` / `Ctrl+T` | Create a new note and open it in a new tab |
| `Meta+W` / `Ctrl+W` | Close the active tab |

Shortcuts are registered via `useAppKeyboardShortcuts` and wired to `createAndOpenNote` and `handleCloseActiveTab`.

## Tab Opening Triggers

Tabs can be opened from three entry points:

| Source | File | Flow |
|--------|------|------|
| **Home grid** | `components/NoteCard.tsx` | Tapping a note card calls `openTab(noteId, title)` + `router.push(/editor?id=...)` |
| **Keyboard shortcut** | `hooks/useCreateAndOpenNote.ts` | `Meta+T` creates a new note, calls `openTab(newId, title)` + `router.push` with `isNew: "true"` |
| **Wiki links** | `components/editor/HybridEditor.tsx` | Clicking `[[Some Note]]` resolves/creates the note ID, calls `openTab(noteId, title)` + `router.push` |

## URL ↔ Store Synchronization

Tabs and the router URL are kept in sync through **explicit navigation calls** (not auto-synced):

| Action | Store Call | Router Call |
|--------|-----------|-------------|
| Open tab | `openTab(noteId, title)` | `router.push(/editor?id=...)` |
| Switch tab | `activateTab(tabId)` | `router.replace(/editor?id=...)` |
| Close tab | `closeTab(tabId)` | `router.replace(/editor?id=${nextActive.noteId})` or `router.replace("/")` |

The URL param `id` is the **source of truth** for which note the editor displays. The tab store provides the tab list and active tab selection logic.
